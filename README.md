# По второму вопросу
## Потоковое хранение больших вложений (~100 МБ) в веб-формах

Если у нас тысячи отправлений и у каждого может быть несколько больших файлов, хранить их в памяти не получится. Решение — читать файлы потоками и сразу писать на диск или облако.

## Архитектура

1. **Frontend (Vue / JS)**  
   - Каждый файл отправляется отдельным запросом (`POST /upload`) целиком или кусками.  

2. **Backend (ASP.NET Core)**  
   - Контроллер читает файл из `Request.Body` кусками (`streaming`) и сразу пишет на диск.  
   - Не создаётся лишняя копия файла в памяти (как с IFormFile).  
   - После полной загрузки сохраняются метаданные в БД (`Attachments`).

3. **Файловое хранилище**  
   - Папки: `/storage/{submissionId}/{fileId}`  
   - Каждый файл + метаданные (`.meta.json`) отдельно для ускорения поиска, а не в одной директории

4. **База данных**  
   - Таблицы:
     - `Submissions` — данные отправлений
     - `Files` или чаще `Attachment` — метаданные файлов
   - **Сами файлы не хранятся в БД**, только метаданные.

## REST API

- `POST /api/submissions/{id}/files` — загрузка одного файла потоками  
- `GET /api/files/{fileId}` — скачивание файла  
- `GET /api/submissions/{id}` — список файлов отправления
